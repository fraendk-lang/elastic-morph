# Fremdtest — Phase 1 Validierung

**Ziel:** Eine Person **ohne Erklärung** exportiert ein Visual in **unter 10 Minuten**.

---

## Tester-Profil

- Musiker/in, Producer/in oder Creator/in
- **Kein** Entwickler
- Hat Elastic Morph **noch nie** gesehen
- Chrome oder Edge auf Laptop/Desktop (ideal)

---

## Link an Tester schicken

```
https://elasticmorph.app
```

*(Alternativ direkt: https://elasticmorph.app/elastic-morph.html)*

---

## Script (Wortlaut — nichts hinzufügen)

> „Mach bitte aus diesem Song ein Visual für Instagram oder YouTube — ohne meine Hilfe.  
> Du darfst alles in der App ausprobieren, aber ich erkläre nichts.“

Optional Track: Tester darf **eigenen MP3** laden **oder** den eingebauten Demo-Track nutzen.

---

## Du beobachtest (still, mitnotieren)

| # | Frage | ☐ Ja / ☐ Nein / Notiz |
|---|--------|----------------------|
| 1 | Findet **Demo starten** allein? | |
| 2 | Versteht **Look wählen** (Swipe/Karten)? | |
| 3 | Findet **Export** (● oder HQ)? | |
| 4 | Bekommt eine **Datei** (Download)? | |
| 5 | Erstes Visual — **Wow** oder **hä**? | |
| 6 | Wo bricht ab? (Schritt 1 / 2 / 3) | |
| 7 | Zeit bis Export: _____ Min | |

---

## Erfolg / Fail

| Ergebnis | Nächster Schritt |
|----------|------------------|
| ✅ MP4 in **< 10 Min** | Phase 1 bestanden → Launch-Inhalte |
| ⚠️ Export OK, aber > 10 Min | UX vereinfachen, 2. Tester |
| ❌ Scheitert an Schritt X | Bug/UX fixen, **nicht** launchen |

---

## Nach dem Test (Tester fragen)

1. „Was war unklar?“
2. „Würdest du das einem Kollegen schicken?“
3. „Was hat dich am meisten überrascht?“

## Export-Hilfe (nur wenn Tester komplett stuck)

Erst nach **8+ Minuten** ohne Fortschritt — **minimal** eingreifen:

> „Unten Format **9:16** wählen, dann den runden **● Export** Button.“

Das zählt als **teilweise Fail** — trotzdem notieren.
