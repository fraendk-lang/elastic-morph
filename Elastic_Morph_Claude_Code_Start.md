## 2. Grundidee

**Elastic Morph** ist eine lokale Audio-Visual-App, die Musik nicht nur beat-synchron visualisiert, sondern daraus ein lebendes visuelles System erzeugt.

Die Visuals sollen sich permanent mit der Musik weiterentwickeln. Sie sollen wachsen, sich verdichten, zerfallen, zurückkehren, mutieren und sich an frühere visuelle Zustände erinnern.

Es soll ausdrücklich **kein einfacher Beat-Visualizer** werden.

Der zentrale Gedanke lautet:

> Elastic Morph turns music into a living visual organism.

Deutsch:

> Elastic Morph verwandelt Musik in einen lebenden visuellen Organismus.

---

## 3. Zielgruppe

Elastic Morph soll mehrere Zielgruppen ansprechen:

### VJs

- Live-Visuals für Bühne, Club, Projektion und Performance
- Fullscreen-Ausgabe
- gut lesbare Performance-Regler
- später MIDI-ready denkbar
- lange, nicht statische Visual-Entwicklung

### Content Creator

- schnelle Visuals für YouTube, Reels, Shorts und Social Media
- Formate 16:9, 9:16, 1:1
- Cover-Motion und kurze Loop-Exports
- hochwertige, wiedererkennbare Ästhetik

### Musiker / Producer

- visuelle Identität pro Track, EP oder Künstlerprojekt
- Audio-Upload als zentraler Startpunkt
- Export für Musikvideos, Social Clips oder Visual Loops

### AI Video / Visual Creator

- kontrollierbare Alternative zu generischen AI-Videos
- keine API-Kosten
- lokales, reproduzierbares System

## 4. Abgrenzung zu klassischen Visualizern

Elastic Morph soll nicht nur auf Bass, Kick oder Lautstärke reagieren.

Klassischer Visualizer:

```text
Audio Input -> Beat Reaction -> Pulsing Shape
```

Elastic Morph:

```text
Audio Input -> Song Map -> Visual DNA -> Morph Memory -> Evolving Visual Organism
```

Der wichtigste Unterschied:

> Die Visuals reagieren nicht nur auf Musik. Sie entwickeln sich mit der Musik.

## 5. Zentrale Produktidee: Visual DNA

Jeder Track bekommt eine eigene **Visual DNA**.

Diese Visual DNA definiert, wie sich das Visual verhält, bewegt und verändert.

### Bestandteile der Visual DNA

#### 1. Form-Verhalten

Beispiele:

- organisch
- geometrisch
- flüssig
- schwer
- schwebend
- zerbrechlich
- fragmentiert
- architektonisch

#### 2. Bewegungscharakter

Beispiele:

- langsame Drift
- rhythmische Impulse
- Atembewegung
- Mikro-Zittern
- Kamera-Sog
- Rotation
- Expansion
- Kontraktion

#### 3. Farbentwicklung

Nicht nur feste Farben, sondern Entwicklung über Zeit:

- dunkel zu offen
- kalt zu warm
- monochrom zu spektral
- tiefblau zu amber
- schwarz zu gläsernem Licht

#### 4. Materialität

Beispiele:

- Glas
- Nebel
- Staub
- Flüssigkeit
- Metall
- Korn
- Lichtfäden
- Zellstruktur
- digitale Partikel

#### 5. Song-Gedächtnis

Die App soll sich frühere visuelle Zustände merken.

Wenn ein musikalisches Motiv später zurückkehrt, darf auch ein visuelles Motiv zurückkehren — aber verändert, größer, beschädigter, ruhiger oder komplexer.

## 6. Hauptfeature: Morph Memory

**Morph Memory** ist das wichtigste Unterscheidungsmerkmal von Elastic Morph.

Die App soll frühere visuelle Zustände speichern und sie später transformiert wieder einführen.

Beispiel:

```text
00:00   kleiner Lichtkern entsteht
00:45   der Kern wird flüssig und beginnt zu wachsen
01:30   Form öffnet sich und wird dichter
02:10   Break: alles zieht sich zusammen
02:50   Rückkehr: Anfangsmotiv erscheint wieder, aber komplexer
03:40   Outro: die Form zerfällt und löst sich auf
```

Das Ziel:

> Das Visual soll wirken, als hätte es den Song verstanden.

---

## 7. Visuelle Dramaturgie

Die Visuals sollen eine Entwicklung über den gesamten Track haben.

Mögliche interne Struktur:

```text
Seed -> Growth -> Tension -> Collapse -> Return -> Dissolve
```

Oder in UI-freundlicher Sprache:

```text
Birth -> Grow -> Break -> Morph -> Return -> Fade
```

Diese Phasen können automatisch aus der Audioanalyse erzeugt werden oder zunächst als einfache Zeitabschnitte simuliert werden.

## 11. Wichtige Regler und Bedeutung

### Morph Amount

Wie stark die Grundform ihre Gestalt verändert.

### Memory

Wie stark frühere Formen später zurückkehren.

### Mutation

Wie unvorhersehbar oder experimentell die Entwicklung ist.

### Gravity

Wie schwer oder schwebend das Visual wirkt.

### Pulse Depth

Wie stark die Musik direkt in Bewegung, Skalierung oder Dichte eingreift.

### Density

Wie voll oder leer das Bild ist.

### Color Drift

Wie stark sich die Farben über Zeit entwickeln.

### Camera Drift

Wie stark die virtuelle Kamera schwebt, zoomt oder rotiert.

### Organic Motion

Wie lebendig und natürlich die Bewegung wirkt.

### Identity Lock

Wie stark die visuelle Identität geschützt bleibt.

Bei hohem Identity Lock bleibt das Visual stilistisch konsistent.

Bei niedrigem Identity Lock darf es stärker mutieren.

## 13. Audioanalyse für MVP

Für die erste Version reicht eine einfache, robuste Analyse.

### Analysewerte

```text
bass
mids
highs
loudness
transient estimate
energy average
time position
track progress
```

### Mögliche Frequenzbereiche

```text
Bass: 20–160 Hz
Mids: 160–2500 Hz
Highs: 2500–12000 Hz
```

### Song Progress

```text
progress = currentTime / duration
```

### Simple Song Map

Für MVP kann eine grobe automatische Einteilung reichen:

```text
0–15%    Birth / Intro
15–35%   Growth
35–55%   Tension
55–70%   Break / Collapse
70–90%   Return / Morph
90–100%  Fade / Outro
```

Später kann echte Struktur-Erkennung ergänzt werden.

## 14. Visual Engine für MVP

Die erste Visual Engine kann mit folgenden Elementen arbeiten:

- Partikel
- morphende Zentralform
- Trails
- radialer Flow
- Noise Field
- Audio-reactive displacement
- Farbdrift
- Kamera-Zoom
- Rotation
- leichte Unschärfe / Glow
- Layer mit Transparenz

### Grundverhalten

Das Visual startet aus einem Seed.

Mit steigender Energie:

- wächst die Form
- steigt die Dichte
- nimmt die Bewegung zu
- öffnet sich das Bild

In Breaks:

- zieht sich die Form zusammen
- Bewegung wird langsamer
- Dichte sinkt
- Farben werden dunkler

Im Return:

- frühere Formen kommen zurück
- aber mit veränderter Größe, Farbe oder Struktur

Im Outro:

- Formen lösen sich auf
- Partikel verschwinden
- Bewegung wird langsamer

## 15. Morph Memory — technische Idee

Für den MVP kann Morph Memory einfach gelöst werden.

### Vorschlag

Speichere alle X Sekunden einen visuellen Zustand:

```js
memorySnapshots.push({
  time,
  seed,
  colorState,
  shapeState,
  density,
  rotation,
  particlePositionsSample
});
```

Später im Track können frühere Snapshots wieder eingemischt werden.

Beispiel:

```js
if (progress > 0.7 && memory > 0.5) {
  blendCurrentStateWith(memorySnapshots[0], memoryAmount);
}
```

Ziel ist nicht perfekte Physik, sondern das Gefühl visueller Erinnerung.

## 18. Visual Identity Lock

Identity Lock schützt den Stil.

### Verhalten

Hoher Wert:

- Farben bleiben nahe am Preset
- Formen bleiben stabiler
- Mutation ist begrenzt
- Visual wirkt konsistent

Niedriger Wert:

- mehr Zufall
- stärkere Verformung
- mehr Farbwechsel
- experimenteller Look

Das ist wichtig, weil viele generative Tools beliebig wirken.

Elastic Morph soll dagegen eine wiedererkennbare visuelle Handschrift ermöglichen.

## 26. Wichtigster Qualitätscheck

Der Prototyp ist nur dann gelungen, wenn man beim Abspielen eines Songs spürt:

1. Das Visual reagiert auf die Musik.
2. Das Visual verändert sich über die Zeit.
3. Das Visual bleibt stilistisch zusammenhängend.
4. Frühere Formen können später wieder auftauchen.
5. Es wirkt nicht wie ein normaler Loop.

Wenn diese fünf Punkte erfüllt sind, ist der Kern von Elastic Morph getroffen.
